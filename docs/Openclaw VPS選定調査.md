# OpenClawを月額$20以下のVPSで動かす最適解

**OpenClawはクラウドAPI経由で動かす限り非常に軽量なアプリケーションであり、月額$5〜$15のVPSで十分に運用できる。** 最もコスパが高いのは**Hetzner Cloud（CX23/CX33）**で、無料を求めるなら**Oracle Cloud Always Free**が圧倒的。以下、プロジェクトの要件分析からVPS比較、最終推奨まで詳しく解説する。

---

## OpenClawとは何か

OpenClaw（旧称Clawdbot / Moltbot）は、**ローカル環境で動作する自律型AIエージェント／パーソナルAIアシスタント**のオープンソースプロジェクトだ。WhatsApp、Telegram、Slack、Discord、Signal、LINE、Matrixなど**20以上のメッセージングプラットフォーム**に接続し、シェルコマンド実行、ブラウザ自動操作、メール管理、カレンダー操作、ファイル操作などを自律的にこなす。

GitHubリポジトリは **github.com/openclaw/openclaw** で、2025年11月の初版リリースからわずか数ヶ月で**約32.7万スター**を獲得した、GitHub史上最速で成長したプロジェクトのひとつ。TypeScriptで書かれ、MITライセンスで公開されている。Anthropic Claude、OpenAI GPT、DeepSeekなど主要LLMにAPIキー経由で接続するモデル非依存設計で、Ollama/LM Studioを使ったローカルモデルにも対応する。

## システム要件は「Gatewayモード」なら極めて軽い

OpenClawには2つの動作モードがあり、**VPS運用ではGatewayモード（クラウドAPI利用）が前提**となる。このモードではLLMの推論はクラウド側で行われ、サーバーはNode.jsのゲートウェイプロセスを実行するだけなので、必要リソースは最小限だ。

| 要件 | 最低 | 推奨（個人利用） | 本番（マルチエージェント） |
|------|------|-----------------|------------------------|
| **CPU** | 1〜2 vCPU | **2〜4コア** | 8スレッド以上 |
| **メモリ** | 2 GB（これ以下でクラッシュ） | **4 GB** | 8〜16 GB |
| **ストレージ** | 256 GB SSD | 40〜80 GB NVMe | 512 GB以上 |
| **GPU** | **不要** | 不要 | 不要 |
| **ネットワーク** | 安定したインターネット接続 | 同左 | 同左 |

**RAMが最も重要な要素**であり、Node.jsのV8エンジンは1 GB以下で確実にクラッシュする。個人利用・開発用なら**4 GB RAMが快適ライン**で、2 GBでもギリギリ動作する。CPUはほぼボトルネックにならない。GPUは一切不要。ローカルモデルを動かす場合のみ16 GB以上のRAMとGPUが必要になるが、VPS運用では現実的ではない。

**Docker対応は万全**で、Dockerfile、docker-compose.yml、ワンコマンドセットアップスクリプト（`docker-setup.sh`）が同梱されている。DigitalOcean・Hetzner・Hostinger向けの1-Clickデプロイも用意されており、推奨OSは**Ubuntu 22.04 / 24.04 LTS**だ。

## 同じ$12でこれだけ差がつく

この比較で際立つのは、**同価格帯でのスペック格差**だ。月額約$12で見ると、DigitalOceanとLinodeは「1 vCPU / 2 GB RAM / 50 GB SSD」なのに対し、**Hetzner CX43は「8 vCPU / 16 GB RAM / 160 GB NVMe」を約$11で提供**している。Contaboも$7.95で「6 vCPU / 12 GB RAM / 100 GB NVMe」と、米国系プロバイダーの6〜8倍のスペックを同価格で実現している。

DigitalOceanやLinodeはマネージドサービスのエコシステム、ドキュメントの充実度、サポート品質で優位に立つが、**OpenClawのようなシンプルなDocker／Node.jsアプリを動かすだけなら、この差額は正当化しにくい**。ただし、新規アカウントならDigitalOceanは**$200の無料クレジット（60日有効）**、Linodeは**$100の無料クレジット（60日有効）**を提供しており、試用期間としては魅力的だ。

なお、Hetznerは**2026年4月1日から値上げ**を予定しており、CX23は€3.49→€4.49、CX33は€5.49→€8.49になる。それでも競合と比較して圧倒的な価格優位性は維持される。また、CX/CAXシリーズはEU（ドイツ・フィンランド）リージョン限定であり、米国やアジアからのレイテンシが大きくなる点には注意が必要だ。

## Oracle Cloudの「永久無料」は本物だが落とし穴もある

Oracle Cloud Always Free tierは、**4 ARM OCPU＋24 GB RAMを月額$0で永久利用可能**という他社を圧倒するオファーだ。OpenClawの要件（4 GB RAM推奨）を大幅に超えるスペックが完全無料で手に入るため、一見すると議論の余地なく最適に見える。

しかし、重大な注意点が2つある。第一に、**人気リージョンではARM（A1）インスタンスの空き容量が慢性的に不足**しており、作成時に「Out of host capacity」エラーが頻発する。何度もリトライするか、マイナーなリージョンを選ぶ必要がある。第二に、**アイドル状態のインスタンスは回収される可能性**がある。CPU・ネットワーク・メモリの使用率がいずれも7日間連続で20%未満だと、Oracleがインスタンスを停止・回収する場合がある。OpenClawはメッセージ待受時にアイドル状態になりがちなので、cronジョブなどで定期的に負荷をかける工夫が必要だ。

無料tier専用アカウントにはSLAやOracleサポートの対象外という制約もある。とはいえ、**個人開発・学習用途でコストゼロを最優先するなら、これ以上の選択肢はない**。

## ベスト3の結論と推奨構成

### 🥇 第1推奨：Hetzner Cloud CX33（月額 ~$6.43 / €5.49）

**4 vCPU / 8 GB RAM / 80 GB NVMe / 20 TB転送量**。OpenClawの推奨要件を余裕で満たし、ブラウザ自動操作やマルチスキル実行時のメモリスパイクにも8 GBなら安心して対応できる。20 TBの転送量は事実上無制限と同義。NVMe SSD（RAID10）による高速ストレージ、Docker 1-Clickデプロイ対応、そして**月額わずか約$6.50**。Hetznerの信頼性とパフォーマンスは定評があり、**コスパ最強の有料選択肢**だ。4月以降も€8.49（約$10）で、依然として最もバランスが良い。EUリージョン限定だが、OpenClawはAPIコール経由でLLMと通信するため、多少のレイテンシは実用上問題にならない。

### 🥈 第2推奨：Oracle Cloud Always Free（月額 $0）

**4 ARM OCPU / 24 GB RAM / 200 GB ストレージ / 10 TB転送量**。コストゼロで最大スペックという唯一無二の選択肢。ARMアーキテクチャだがOpenClawはNode.jsベースなので完全に動作し、Docker公式のARM対応イメージも利用可能。インスタンス確保の困難さとアイドル回収リスクを許容できるなら、試す価値は極めて高い。まずOracle Cloudで無料運用を試み、問題があればHetznerに移行するという段階的アプローチが最も合理的だ。

### 🥉 第3推奨：Contabo Cloud VPS 10（月額 $4.95）

**4 vCPU / 8 GB RAM / 75 GB NVMe / 転送量無制限**。Hetznerとほぼ同スペックを$5以下で提供し、**東京DCが選択可能**な点が大きな差別化要素。日本からの低レイテンシアクセスを重視する場合、Contaboは有力な選択肢になる。ただし、Hetznerと比較してサポートの応答速度やCPUパフォーマンスの安定性でやや劣るという評価がある点には留意したい。

### 構築の手順イメージ

いずれのVPSを選んでも、セットアップは以下の3ステップで完了する：

1. Ubuntu 24.04 LTSでVPSを作成し、DockerとDocker Composeをインストール
2. `git clone https://github.com/openclaw/openclaw.git && cd openclaw`
3. `./docker-setup.sh` を実行（対話型ウィザードでLLM APIキー設定・チャネル接続）

PM2やsystemdでプロセスの自動再起動を設定し、`~/.openclaw/`ディレクトリのバックアップを定期的に取得すれば、安定した24/7運用が実現できる。なお、**サーバー費用よりもLLM APIの利用料金（月$10〜数百ドル）の方がはるかに大きなコスト要因**になる点は、予算計画に必ず織り込んでおくべきだ。
