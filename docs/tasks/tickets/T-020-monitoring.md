# T-020: 監視・アラート設定

## 概要
パイプラインの実行失敗や異常を検知し、通知する仕組みを構築する。

## やること
- 実行結果の監視方法の決定（以下から選択）:
  - GitHub Actions でサイト更新の有無をチェック
  - VPS 上で簡易ヘルスチェックスクリプト
  - Cloudflare Pages のデプロイ履歴確認
- 通知先の決定（メール、Slack、Discord 等）
- 「記事が0件だった日」の検知と通知
- setup-token 期限切れの事前アラート

## 完了条件
- パイプライン失敗時に通知が届く
- 2日以上記事が更新されない場合にアラートが出る
