# T-013.5: OpenClaw Gateway 設定

## 概要
Hetzner VPS 上に OpenClaw Gateway をインストールし、MiniMax M2.5 を LLM プロバイダとして設定する。

## 依存
- T-013（Hetzner VPS セットアップ）

## やること

### 1. OpenClaw インストール
```bash
curl -fsSL https://get.openclaw.ai | bash
openclaw onboard --install-daemon
```

### 2. MiniMax M2.5 をカスタムプロバイダとして設定
`~/.openclaw/openclaw.json` に以下を追加:
```json
{
  "models": {
    "providers": {
      "minimax": {
        "baseUrl": "https://api.minimax.chat/v1/openai",
        "apiKey": "${MINIMAX_API_KEY}",
        "api": "openai-completions",
        "models": {
          "minimax-m2.5": {
            "contextWindow": 197000,
            "maxTokens": 16384
          }
        }
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "minimax/minimax-m2.5"
      }
    }
  }
}
```

### 3. ワークスペース設定
- **SOUL.md**: エージェントの役割定義（日本語テックニュース収集・要約エージェント）
- **AGENTS.md**: ツール許可設定（bash, read, write, edit）とセキュリティルール

### 4. Gateway 起動確認
```bash
systemctl --user status openclaw-gateway
openclaw gateway status
```

### 5. 動作テスト
テストメッセージを送信し、MiniMax M2.5 で応答が返ることを確認。

## 完了条件
- Gateway が systemd ユーザーサービスとして稼働中
- MiniMax M2.5 でエージェントが応答する
- `openclaw gateway status` が healthy を返す

**ステータス**: ✅ 完了（2026-03-22）

## 完了メモ
- OpenClaw 2026.3.13 インストール済み、Gateway は systemd ユーザーサービスとして稼働（enabled, port 18789）
- MiniMax M2.5 を OpenAI互換エンドポイント (`https://api.minimax.io/v1`) で接続
- Coding Plan Starter の APIキー（`sk-cp-*`）は Anthropic互換エンドポイントでは billing error になるため、`api: "openai-completions"` で設定
- systemd に EnvironmentFile で `.env` を読み込むオーバーライド追加（`~/.config/systemd/user/openclaw-gateway.service.d/env.conf`）
- SOUL.md, AGENTS.md 等のワークスペースファイルは `~/.openclaw/workspace/` に配置済み
