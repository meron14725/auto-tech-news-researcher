# T-019: エラーハンドリング・ログ整備

## 概要
パイプライン全体のエラーハンドリングを強化し、障害時の原因特定を容易にするログ基盤を整備する。

## やること
- `run-daily.sh` の各ステップでのエラー検知と適切な終了
- 1ソースの障害時に他ソースで処理を継続する仕組み
- ログ出力の整備:
  - 実行開始・終了時刻
  - 各ソースの取得件数
  - フィルタリング結果（採用数/除外数）
  - エラー発生箇所と内容
- ログファイルのローテーション（古いログの自動削除）

## 完了条件
- 1ソースがダウンしても他ソースの記事は正常に処理される
- エラー発生時にログから原因が特定できる
- ログが日付別に保存される
