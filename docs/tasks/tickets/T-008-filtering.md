# T-008: 記事フィルタリング・スコアリング機能

**ステータス**: ✅ 完了（2026-03-15）

## 概要
収集した記事に対して Claude がテック分野での重要度を評価し、`interest_score` を付与する。スコア7以上の記事のみ採用する。

## やること
- 各記事に対して interest_score（1〜10）を付与する評価基準の定義
- 評価基準: テーマの新規性、技術的インパクト、コミュニティの関心度など
- スコア7未満の記事を除外するフィルタリングロジック
- スキル内での評価プロンプト設計

## 完了条件
- 全記事に interest_score が付与される
- スコア7未満の記事が出力から除外される
- 1日あたりの採用記事数が適切な範囲（5〜15件程度）に収まる

## 完了メモ
- SKILL.md にスコアリング基準を定義:
  - AI/LLM/ML: +3、言語/FW: +2、DevOps/インフラ: +2、セキュリティ: +2、OSS/ツール: +1
  - HN高ポイント(100+): +1、Zennトレンド: +1
- score >= 7 のみ採用、最大15件に制限
