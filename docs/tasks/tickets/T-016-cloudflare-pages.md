# T-016: Cloudflare Pages デプロイ設定

## 概要
GitHub リポジトリと Cloudflare Pages を連携し、main ブランチへの push をトリガーに Hugo ビルド → 自動デプロイする。

## 手順

### 1. Cloudflare アカウント準備
1. [Cloudflare ダッシュボード](https://dash.cloudflare.com/) にログイン（未登録なら無料アカウント作成）
2. Workers & Pages → 概要 を開く

### 2. Pages プロジェクト作成
1. 「アプリケーションを作成」→「Pages」→「Git に接続」
2. GitHub アカウントを連携
3. リポジトリ `auto-tech-news-researcher` を選択
4. ビルド設定:

| 設定項目 | 値 |
|---------|------|
| プロダクションブランチ | `main` |
| フレームワーク | Hugo |
| ビルドコマンド | `hugo` |
| 出力ディレクトリ | `public` |

5. 環境変数を追加:

| 変数名 | 値 | 説明 |
|--------|------|------|
| `HUGO_VERSION` | `0.157.0` | ローカルと同じバージョン |

6. 「保存してデプロイ」をクリック

### 3. デプロイ確認
1. ビルドログで Hugo ビルドが成功していることを確認
2. 生成された URL（`xxx.pages.dev`）にアクセス
3. サイトが表示され、サンプル記事が見えることを確認

### 4. Git サブモジュール対応
Cloudflare Pages はサブモジュールを自動取得する。もし失敗する場合:
- ビルドコマンドを以下に変更:
  ```
  git submodule update --init --recursive && hugo
  ```

### 5. カスタムドメイン設定（任意）
1. Cloudflare Pages → カスタムドメイン → ドメインを追加
2. DNS レコードが自動設定される（Cloudflare で管理している場合）
3. `hugo.toml` の `baseURL` をカスタムドメインに更新:
   ```toml
   baseURL = 'https://your-domain.com/'
   ```
4. 変更をコミット & プッシュ → 再デプロイ

### 6. デプロイ設定の確認
- ビルドが main push 時に自動トリガーされることを確認
- PR 作成時にプレビューデプロイが作られることを確認（デフォルト有効）

## 完了条件
- main に push すると自動でビルド＆デプロイが実行される
- デプロイされたサイトにブラウザからアクセスできる
- Hugo ビルドがエラーなく完了する
- サンプル記事ページが正しく表示される

## 無料枠の制限
| 項目 | 制限 |
|------|------|
| ビルド回数 | 500回/月 |
| ファイル数 | 20,000ファイル |
| ファイルサイズ | 25MB/ファイル |
| 帯域 | 無制限 |

毎日1回デプロイ（月30回）なので十分余裕あり。
